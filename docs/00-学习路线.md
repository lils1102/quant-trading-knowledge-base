# 00 - 学习路线

本章回答一个核心问题：**应该按什么顺序学习量化交易？**

量化交易不是“学会 Python、调一个模型、自动下单”这么简单。更准确地说，它是一套把金融市场、数据处理、统计研究、交易系统和风险管理组合起来的工作流。

## 一句话总览

```text
看懂市场
  -> 认识交易品种
  -> 提出策略假设
  -> 获取和清洗数据
  -> 回测验证
  -> 风控约束
  -> 模拟交易
  -> 小规模实盘
  -> 监控和复盘
```

每一步都不能跳。跳过市场知识，容易写出看似聪明但无法交易的策略；跳过数据清洗，容易得到虚假的回测收益；跳过风险控制，实盘中一次异常就可能造成严重损失。

## 第 1 阶段：理解金融市场在做什么

先弄清楚几个基础问题：

- 股票为什么会涨跌？
- 债券为什么会受利率影响？
- ETF 为什么既像股票又像基金？
- 期货为什么需要保证金？
- 期权为什么不只看方向，还要看波动率和到期时间？
- 市价单、限价单、止损单有什么区别？
- 流动性差时，为什么回测收益可能无法兑现？

这一阶段的目标不是预测行情，而是知道市场如何运行、订单如何成交、风险从哪里来。

## 第 2 阶段：用“假设”理解策略

量化策略不是神秘公式，而是一句可以被检验的假设。

例子 1：

```text
过去一段时间涨得更多的资产，未来短期内可能继续上涨。
```

这属于动量思路。

例子 2：

```text
两个长期相关的资产，短期价差异常扩大后，未来可能重新靠近。
```

这属于均值回归或配对交易思路。

一个合格的策略假设至少要回答：

1. 这个规律为什么可能存在？
2. 收益来源可能是什么？
3. 交易成本会不会吃掉收益？
4. 什么情况下这个规律会失效？
5. 发生亏损时如何退出？

如果只能解释为“历史曲线好看”，还不能算可靠策略。

## 第 3 阶段：建立数据意识

量化研究里，很多错误不是模型造成的，而是数据造成的。

需要特别注意：

- **复权**：股票分红、拆股后价格需要处理，否则收益率会失真。
- **幸存者偏差**：只看今天还存在的股票，会高估历史表现。
- **未来函数**：使用当时还不知道的数据，会让回测虚高。
- **交易日历**：不同市场的开盘、休市和结算时间不同。
- **数据版本**：财报、指数成分、宏观数据可能被修正，研究时要记录版本。

刚开始做研究时，建议先使用日线数据和流动性较好的标的，等流程稳定后再处理分钟线、Tick、订单簿等高频数据。

## 第 4 阶段：做一个诚实的回测

回测不是为了证明自己正确，而是为了尽早暴露策略缺陷。

一个基础回测至少包括：

- 买入和卖出规则
- 持仓和调仓规则
- 手续费
- 滑点
- 停牌或无法成交处理
- 样本内和样本外区分
- 最大回撤、波动率、胜率、盈亏比、换手率等指标

不要只看年化收益率。高收益但大回撤、高换手、低容量、强依赖单一市场阶段的策略，在实盘中往往很脆弱。

## 第 5 阶段：先风控，再收益

真实交易中，最重要的不是赚得快，而是亏损时不失控。

基础风控包括：

- 单笔最大亏损
- 单策略最大仓位
- 单资产最大仓位
- 总杠杆上限
- 单日最大亏损
- 最大回撤限制
- 流动性限制
- 异常行情暂停交易
- 数据中断暂停交易
- 订单失败报警

没有风控的盈利策略，本质上只是还没有遇到足以摧毁它的市场环境。

## 第 6 阶段：再考虑实盘系统

实盘系统至少包括：

```text
行情模块 -> 策略模块 -> 风控模块 -> 订单模块 -> 交易接口 -> 持仓账户模块 -> 日志监控模块
```

工程上要做到：

- 每一笔信号可以追溯来源
- 每一笔订单可以查询状态
- 每一次持仓变化都有记录
- 出现异常时可以自动暂停或降级
- 策略版本、参数、数据版本都能复现

## 第 7 阶段：谨慎使用机器学习和 AI

机器学习适合用于：

- 因子组合
- 非线性预测
- 文本和新闻处理
- 财报和研报摘要
- 异常检测
- 参数筛选辅助

但它最容易出现的问题是：

- 过拟合
- 标签泄漏
- 样本外失效
- 训练环境和实盘环境不一致
- 模型解释不足

AI/Agent 可以辅助读资料、写代码、整理研究假设，但交易决策仍应受数据验证、人工审查和风控规则约束。

## 推荐推进顺序

### 阶段 A：市场与数据分析

- 学股票、ETF、债券、期货、期权的基本概念
- 学会读取表格数据、计算收益率、均线、波动率
- 能解释价格、成交量、流动性和订单类型

### 阶段 B：基础策略与回测

- 实现均线策略
- 实现动量策略
- 实现均值回归策略
- 加入手续费和滑点
- 学会评估最大回撤、夏普比率和换手率

### 阶段 C：因子研究

- 理解什么是因子
- 做分组回测
- 计算 IC、Rank IC
- 学会去极值、标准化、中性化
- 检查因子衰减和失效

### 阶段 D：组合与风控

- 理解相关性
- 学会仓位管理
- 建立风险预算
- 制作上线前检查清单

### 阶段 E：模拟交易与实盘验证

- 接入模拟交易接口
- 建立日志和监控
- 小规模验证
- 定期复盘和迭代

## 读完本仓库后应该能回答

- 什么是可交易的策略假设？
- 如何判断回测是否可信？
- 实盘交易系统至少需要哪些模块？
- 如何识别常见研究陷阱？
- AI 在量化研究中应该放在什么位置？
