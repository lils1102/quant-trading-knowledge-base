# 09 - 数据工程

本章目标：理解量化研究为什么高度依赖数据工程。策略看起来是在研究价格和模型，实际首先是在研究数据是否可靠、是否及时、是否可复现。

## 1. 为什么数据工程重要

量化交易的输入是数据，输出是交易决策。如果数据错了，后面的模型、回测和风控都会跟着错。

常见问题包括：

- 价格没有复权，收益率计算错误。
- 财报数据用了公告前还不知道的信息。
- 指数成分只用今天的成分，产生幸存者偏差。
- 不同市场时区没有对齐。
- 宏观数据后来被修正，却被当成当时已知数据。
- Tick 数据缺失或重复，导致高频信号失真。

数据工程的目标是：让数据可获取、可校验、可追溯、可复现。

## 2. 常见数据类型

### 行情数据

包括开盘价、最高价、最低价、收盘价、成交量、成交额、盘口、逐笔成交等。

行情数据用于计算收益率、波动率、趋势、成交活跃度、滑点和流动性。

### 基本面数据

包括财报、估值、盈利能力、现金流、资产负债表、分红、回购等。

基本面数据的重点不是“有没有数据”，而是“当时什么时候知道这条数据”。

### 宏观数据

包括利率、通胀、就业、GDP、汇率、央行资产负债表等。

宏观数据经常有发布时间和修正版本。研究时要区分“当时看到的版本”和“后来修正后的版本”。

### 公司行为数据

包括分红、拆股、配股、并购、退市、停牌、指数纳入和剔除等。

公司行为会影响价格、成交、持仓和回测结果。

### 另类数据

包括新闻、研报、搜索热度、社交媒体、卫星图像、链上数据等。

另类数据通常清洗难度更高，噪声更大，更需要严格验证。

## 3. 数据生命周期

一个完整数据流程可以写成：

```text
数据采集
  -> 原始数据存储
  -> 数据清洗
  -> 数据校验
  -> 复权和对齐
  -> 特征计算
  -> 版本记录
  -> 研究和回测
  -> 实盘监控
```

不要直接覆盖原始数据。原始数据应尽量保留，清洗后的数据另存一份，并记录处理方法。

## 4. 行情数据处理

行情数据常见字段：

```text
datetime, symbol, open, high, low, close, volume, amount
```

需要检查：

- 时间戳是否正确。
- 是否有重复记录。
- 是否有缺失交易日。
- 最高价是否小于最低价。
- 成交量是否为负。
- 停牌日是否被错误填充。
- 分红拆股是否正确复权。

日线研究中，通常要同时保存未复权价格、前复权价格、后复权价格和复权因子，避免以后无法解释结果。

## 5. 财务数据处理

财务数据最容易引入未来函数。

例如，某公司 2024 年年报覆盖的是 2024 年全年，但可能在 2025 年才发布。研究时不能在 2024 年底就使用这份年报。

财务数据应至少记录：

- 报告期
- 公告日期
- 数据发布日期
- 修正日期
- 数据来源
- 字段单位
- 币种

研究中应使用“可获得时间”，而不是“报告期结束时间”。

## 6. 宏观数据处理

宏观数据经常存在修正。

例如，某个经济指标第一次发布后，之后可能被修正。如果直接使用最新修正值去回测过去策略，就可能高估策略表现。

处理宏观数据时要关注：

- 发布时间
- 修正版本
- 发布频率
- 时区
- 节假日
- 数据缺失
- 与交易日的对齐方式

## 7. 存储方式

不同阶段可以使用不同存储方案。

### CSV

适合临时查看和小规模样例，不适合长期维护大规模数据。

### Parquet

适合保存结构化研究数据，压缩率高，读取速度快。

### SQLite / DuckDB

适合本地研究，部署简单，方便做 SQL 查询。

### PostgreSQL / MySQL

适合存储元数据、任务状态、策略配置、交易记录等。

### ClickHouse / 时序数据库

适合大规模行情、日志和高频数据分析。

存储选择不应追求复杂，而应匹配数据规模、查询方式和维护能力。

## 8. 数据校验

数据进入研究流程前，应做自动校验。

常见校验：

- 字段是否齐全。
- 数据类型是否正确。
- 时间戳是否递增。
- 主键是否重复。
- 价格是否异常跳变。
- 成交量是否异常。
- 缺失比例是否超限。
- 不同数据源之间是否一致。

一旦校验失败，应停止后续任务，而不是让错误数据继续进入回测。

## 9. 特征工程

特征工程是把原始数据转换成模型或策略能使用的变量。

常见特征：

- 收益率
- 移动平均线
- 波动率
- 换手率
- 估值指标
- 盈利能力指标
- 行业分类
- 宏观状态变量

特征工程要特别注意：

- 不能使用未来数据。
- 不同频率数据要正确对齐。
- 缺失值处理要有解释。
- 因子计算要记录参数。
- 特征版本要可追溯。

## 10. 数据版本和血缘

数据版本管理回答两个问题：

```text
这次回测用了哪一版数据？
这个数据是怎么处理出来的？
```

建议记录：

- 数据源
- 下载时间
- 原始文件哈希
- 清洗脚本版本
- 参数配置
- 输出文件路径
- 数据覆盖时间范围

如果不能复现数据，就不能复现回测。

## 11. 实践任务

1. 下载一只股票的日线数据，检查是否存在缺失日期和异常价格。
2. 保存未复权价、复权价和复权因子。
3. 找一份财报数据，区分报告期和公告日期。
4. 下载一个宏观指标，观察是否有修正版本。
5. 为一个数据表设计校验规则和字段说明。
