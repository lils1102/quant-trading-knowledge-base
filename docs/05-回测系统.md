# 05 - 回测系统

本章目标：理解回测到底在模拟什么，以及如何判断一个回测结果是否可信。

## 1. 回测是什么

回测是用历史数据模拟策略过去会如何交易。

它的作用不是证明策略一定能赚钱，而是帮助回答：

```text
如果过去按这套规则交易，会发生什么？
收益来自哪里？
最大亏损有多大？
交易成本后还剩多少？
哪些市场环境下容易失效？
```

回测是研究工具，不是未来收益保证。

## 2. 一个最小回测需要什么

最小可用回测至少需要：

- 交易标的
- 历史行情数据
- 买入规则
- 卖出规则
- 仓位规则
- 手续费
- 滑点
- 起始资金
- 评估指标

例如，最简单的均线策略：

```text
当短期均线上穿长期均线时买入
当短期均线下穿长期均线时卖出
每次只持有一个标的
手续费按成交金额的一定比例扣除
```

这个策略简单，但已经包含信号、交易、持仓和成本。

## 3. 向量化回测和事件驱动回测

### 向量化回测

向量化回测通常用表格计算完成，适合快速研究。

优点：

- 速度快
- 代码简洁
- 适合日线和横截面因子研究

缺点：

- 真实成交过程模拟较弱
- 复杂订单、盘口、撮合和异步事件不好处理

### 事件驱动回测

事件驱动回测把市场变化、信号、订单、成交、持仓变化都看成事件。

常见事件：

- 行情事件
- 信号事件
- 下单事件
- 成交事件
- 风控事件
- 账户更新事件

优点：更接近实盘系统。缺点：实现复杂，速度较慢。

## 4. 撮合模拟

真实交易不是“看到价格就一定能按这个价格成交”。撮合模拟要处理：

- 下单时间
- 可成交价格
- 成交数量
- 未成交订单
- 部分成交
- 涨跌停
- 停牌
- 盘口深度
- 订单排队

日线回测中常见简化方式：

```text
今天收盘产生信号
明天开盘成交
成交价 = 明天开盘价 + 滑点
```

这种假设比“今天收盘产生信号并按今天收盘价成交”更保守，因为后者容易引入未来函数。

## 5. 手续费、滑点和冲击成本

交易成本通常包括：

- 佣金
- 印花税或交易税
- 交易所费用
- 买卖价差
- 滑点
- 市场冲击成本
- 融资或融券成本

### 滑点

滑点是预期成交价和实际成交价之间的差异。

例如，回测假设 100 元买入，但实盘平均成交价是 100.08 元，多出来的 0.08 元就是成本。

### 市场冲击

资金量越大，越可能影响市场价格。一个小订单可以轻松成交，但一个大订单可能把价格推高或压低。

忽略交易成本，是很多回测虚高的主要原因。

## 6. 常见回测偏差

### 未来函数

使用了当时还不知道的数据。

例子：用全年财报数据去模拟年初买入，明显不合理。正确做法是按财报实际披露时间使用数据。

### 幸存者偏差

只使用今天仍然存在的股票，忽略历史上已经退市的股票，会高估策略表现。

### 过度拟合

不断调整参数，直到历史曲线最好。结果可能只是拟合了过去噪声。

### 样本选择偏差

只选择策略表现好的市场、行业或时间段，忽略不利样本。

### 忽略流动性

回测中买卖顺利，实盘却无法按足够数量成交。

## 7. 样本内和样本外

开发策略时，可以把数据分成：

- **样本内**：用于提出和调试策略。
- **样本外**：用于检验策略是否有泛化能力。

更稳健的做法是滚动验证：

```text
用过去一段时间训练或选择参数
用下一段时间验证
窗口不断向前滚动
```

时间序列不能随意打乱。金融数据有明显时间顺序，未来数据不能参与过去决策。

## 8. 回测指标

常用指标：

- 总收益率
- 年化收益率
- 年化波动率
- 最大回撤
- 夏普比率
- 卡玛比率
- 胜率
- 盈亏比
- 换手率
- 单笔平均收益
- 最大连续亏损
- 策略容量

不要只看年化收益。更重要的是：收益是否稳定，回撤是否可承受，换手是否过高，策略是否有容量，收益是否集中在少数几笔交易。

## 9. 回测结果如何解释

看到一条漂亮曲线时，先问：

- 是否扣除了真实交易成本？
- 是否有未来函数？
- 是否包含退市和停牌资产？
- 是否做了样本外测试？
- 是否只在某一段行情有效？
- 最大回撤是否可承受？
- 资金容量是否足够？
- 交易频率是否现实？

如果这些问题回答不清楚，回测结果只能作为草稿，不能作为实盘依据。

## 10. 回测系统结构

一个较完整的回测系统可以包括：

```text
数据模块
  -> 日历模块
  -> 策略模块
  -> 信号模块
  -> 组合模块
  -> 风控模块
  -> 订单模块
  -> 撮合模块
  -> 账户模块
  -> 绩效分析模块
```

每个模块应该尽量独立，方便测试和复用。

## 11. 实践任务

1. 实现一个均线策略，先不加成本，再加入手续费和滑点。
2. 比较“当天收盘成交”和“次日开盘成交”的差异。
3. 计算最大回撤、夏普比率和换手率。
4. 把数据分成样本内和样本外，观察表现是否稳定。
5. 找出策略收益最大的 10 笔交易，判断收益是否过度集中。
