# 06 - 实盘交易系统

本章目标：理解一个真实运行的量化交易系统需要哪些模块，以及为什么“能回测”不等于“能实盘”。

## 1. 实盘系统在做什么

实盘系统的任务是把研究阶段产生的策略规则，稳定、可控、可追溯地执行到真实市场中。

它不是简单地把回测代码改成自动下单。真实市场会出现：

- 行情延迟
- 数据缺失
- 订单拒绝
- 部分成交
- 重复下单
- 网络中断
- 接口限流
- 交易所临时规则变化
- 持仓和账户不同步

因此，实盘系统的核心不是“自动交易”，而是“在不确定环境中安全执行交易计划”。

## 2. 总体架构

一个基础实盘系统可以拆成：

```text
行情模块
  -> 数据校验模块
  -> 策略模块
  -> 风控模块
  -> 订单模块
  -> 交易接口
  -> 成交回报模块
  -> 持仓账户模块
  -> 日志监控模块
```

每个模块都应该职责清晰。策略模块只负责产生信号，风控模块决定信号是否允许执行，订单模块负责把交易意图转成真实订单。

## 3. 行情模块

行情模块负责接收市场数据。

常见数据类型：

- 日线数据
- 分钟线数据
- Tick 数据
- 盘口数据
- 成交明细
- 指数和行业数据
- 期货主力合约数据
- 期权波动率和希腊字母数据

行情模块要处理：

- 数据延迟
- 数据重复
- 数据缺失
- 异常价格
- 时间戳错位
- 交易日历
- 时区问题

如果行情数据错误，策略信号就会错误。因此行情进入策略前必须经过校验。

## 4. 策略模块

策略模块负责根据数据产生交易信号。

一个信号至少应该包含：

```text
策略名称
标的代码
方向：买入 / 卖出 / 平仓
目标仓位或目标数量
信号产生时间
使用的数据版本
策略参数版本
```

策略模块不应该直接下单。原因很简单：同一个信号是否能执行，还要经过资金、仓位、风险、流动性和交易规则检查。

## 5. 风控模块

风控模块是实盘系统的安全阀。

常见风控规则：

- 单笔订单金额限制
- 单个标的最大仓位限制
- 单个策略最大仓位限制
- 总杠杆限制
- 单日最大亏损限制
- 最大回撤限制
- 流动性限制
- 黑名单标的限制
- 异常行情暂停交易
- 数据中断暂停交易

风控模块应该独立于策略模块。策略认为可以买，风控仍然可以拒绝。

## 6. 订单模块

订单模块负责把交易意图转换成真实订单。

订单需要记录：

- 订单编号
- 策略编号
- 标的代码
- 买卖方向
- 订单类型
- 委托价格
- 委托数量
- 下单时间
- 状态变化
- 成交数量
- 平均成交价

订单状态可能包括：

```text
待提交
已提交
部分成交
全部成交
已撤单
提交失败
被拒绝
状态未知
```

“状态未知”尤其重要。网络中断时，系统可能不知道订单是否已经到达交易所。此时不能盲目重复下单，必须先查询订单状态。

## 7. 持仓和账户模块

持仓模块记录当前持有哪些资产、数量是多少、成本是多少、浮动盈亏是多少。

账户模块记录：

- 现金余额
- 可用资金
- 冻结资金
- 保证金占用
- 总资产
- 净值
- 当日盈亏
- 累计盈亏

实盘系统必须定期对账：

```text
系统内部持仓
  vs
券商 / 交易所返回持仓
```

如果两边不一致，必须停止交易或进入保护模式。

## 8. 日志和审计

没有日志，就无法复盘和排查事故。

至少要记录：

- 原始行情
- 清洗后的行情
- 策略信号
- 风控判断
- 下单请求
- 订单回报
- 成交回报
- 持仓变化
- 错误和异常
- 系统启动和关闭

日志要能回答：

```text
这笔订单为什么产生？
当时使用了什么数据？
通过了哪些风控？
实际成交价是多少？
有没有发生异常？
```

## 9. 监控和报警

实盘系统必须有监控。

常见监控指标：

- 行情是否正常更新
- 策略是否正常运行
- 订单是否长时间未成交
- 账户和持仓是否同步
- 当日亏损是否接近限制
- 网络连接是否正常
- API 是否报错
- 日志是否持续写入
- 系统资源是否异常

报警方式可以包括：邮件、短信、电话、企业微信、Slack、Telegram 等。关键报警不能只依赖单一渠道。

## 10. 故障处理

常见故障和处理方式：

| 故障 | 处理原则 |
|---|---|
| 行情中断 | 暂停产生新信号 |
| 订单状态未知 | 查询交易接口，不重复下单 |
| 持仓不一致 | 停止交易并人工确认 |
| 当日亏损超限 | 立即停止策略或降仓 |
| 接口限流 | 降低请求频率并进入保护模式 |
| 网络中断 | 保存现场，恢复后先对账 |
| 策略异常 | 停止该策略，不影响其他模块 |

实盘系统应该默认异常一定会发生，提前设计降级和暂停机制。

## 11. 模拟交易和小规模验证

正式扩大资金前，建议经历：

```text
离线回测
  -> 纸面交易
  -> 模拟交易
  -> 小规模真实交易
  -> 稳定运行后逐步扩大
```

每个阶段都要记录：

- 信号是否符合预期
- 成交是否符合预期
- 滑点是否可接受
- 风控是否生效
- 日志是否完整
- 异常是否能被发现
