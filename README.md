# Quant Trading Knowledge Base

面向零基础学习者和程序员的量化交易知识库。

本项目的目标不是收集零散资料，而是帮助你从“完全不懂金融市场”逐步走到“能够设计、回测、验证并谨慎上线一个量化策略”。

> 免责声明：本仓库只用于学习和研究，不构成投资建议。真实交易可能产生亏损，尤其是使用杠杆、期货、期权、保证金或高频交易时。

## 适合谁

- 想系统学习量化交易的小白
- 有编程能力但缺少金融市场知识的软件工程师
- 想从策略研究走向回测、风控、实盘系统的人
- 想用 AI/Agent 辅助量化研究，但不想陷入“黑盒交易”的人

## 学习原则

### 1. 先理解市场，再写代码

量化交易不是“写一个会买卖的程序”这么简单。代码只是执行工具，真正重要的是：

- 为什么这个市场可能存在机会
- 机会来自风险补偿、行为偏差、制度约束，还是数据处理差异
- 机会是否能被交易成本吃掉
- 机会是否会因为太多人使用而消失

### 2. 先能解释，再做回测

一个策略至少要回答：

```text
谁在犯错？
我为什么能赚到钱？
这个规律什么时候会失效？
亏损发生时我如何退出？
```

不能解释的策略，即使回测很好，也要高度怀疑。

### 3. 先控制风险，再追求收益

真实交易中，最重要的不是某一次赚多少，而是：

- 能不能长期活下来
- 大亏时能不能停下来
- 数据出错、网络中断、交易接口异常时会不会失控
- 策略失效后能不能及时发现

### 4. 先小规模验证，再考虑实盘扩大

推荐路线：

```text
手工理解逻辑
  -> 写出可复现研究脚本
  -> 做样本内/样本外测试
  -> 纸面交易或模拟盘
  -> 小资金实盘
  -> 长期监控
  -> 逐步扩大规模
```

## 资料来源原则

本知识库优先参考：

- 监管机构与投资者教育资料：SEC Investor.gov、FINRA、CFTC 等
- 交易所与行业教育资料：CME、OIC 等
- 学术数据与论文：Kenneth French Data Library、Fama-French 因子模型、回测过拟合研究等
- 官方数据接口：SEC EDGAR、FRED 等
- 工程实践：事件驱动系统、日志、监控、风控网关、可复现实验
