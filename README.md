# Quant Trading Knowledge Base

这是一个从基础概念开始、逐步走向策略研究、回测验证、风险控制和实盘系统的量化交易知识库。

本项目不追求堆砌资料，而是把量化交易拆成一条可学习、可实践、可复盘的路径：先理解市场，再提出策略假设，再用数据验证，最后在严格风控下谨慎运行。

> 免责声明：本仓库仅用于学习和研究，不构成投资建议。真实交易可能产生亏损，使用杠杆、期货、期权、保证金或高频交易时风险更高。

## 学习原则

### 1. 先理解市场，再写交易逻辑

量化交易不是“让程序自动买卖”这么简单。代码只是执行工具，真正重要的是：

- 为什么这个市场可能存在机会
- 机会来自风险补偿、行为偏差、制度约束，还是数据处理差异
- 机会是否会被交易成本吃掉
- 机会是否会因为参与者变多而消失

### 2. 先能解释，再做回测

一个策略至少要回答：

```text
谁在承担风险？
收益来源是什么？
这个规律什么时候会失效？
亏损发生时如何退出？
```

不能解释的策略，即使历史曲线很好，也要高度怀疑。

### 3. 先控制风险，再追求收益

真实交易中，最重要的不是某一次赚多少，而是：

- 能不能长期运行
- 出现大亏时能不能停止
- 数据出错、网络中断、接口异常时会不会失控
- 策略失效后能不能及时发现

### 4. 先小规模验证，再扩大规模

推荐路线：

```text
理解交易逻辑
  -> 写出可复现研究脚本
  -> 做样本内/样本外测试
  -> 模拟交易
  -> 小资金实盘
  -> 长期监控
  -> 逐步扩大规模
```

## 资料来源原则

本知识库优先参考：

- 监管机构与投资者教育资料：SEC Investor.gov、FINRA、CFTC 等
- 交易所与行业教育资料：CME、OIC 等
- 学术数据与论文：Kenneth French Data Library、Fama-French 因子模型、回测过拟合研究等
- 官方数据接口：SEC EDGAR、FRED 等
- 工程实践：事件驱动系统、日志、监控、风控网关、可复现实验
